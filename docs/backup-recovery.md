# Backup & Recovery

The platform is **regenerable from source**: everything public is a deterministic function of `data/*.json` + the scripts. Losing `site/` or `reports/` is a non-event — rebuild. The assets that must be protected are the human-owned inputs.

## What to back up (and where)

| Asset | Criticality | Backup |
|---|---|---|
| `data/*.json` (verified records) | **Critical** | Git history (primary). Periodic off-Git export recommended. |
| `config/*.json` | High | Git history. |
| `assets/source/` originals (images) | High | Git-ignored by policy → **must** be backed up separately (they are not in the repo). |
| `data/pipeline.json`, `data/grants.json` | Medium (private) | Git history; keep sensitive detail out and in a private store. |
| Secrets (API keys) | Critical | GitHub Secrets + your password manager. Never in Git. |
| `site/`, `reports/` | None | Regenerated by build; do not back up. |

## Recovery playbook

**Rebuild the whole site:**
```bash
python3 scripts/build.py && python3 tests/test_site.py
```

**Regenerate intelligence + API:**
```bash
python3 scripts/research_os.py --all
```

**Roll back a bad deploy:** revert the offending commit on `main` (or redeploy a known-good commit); Pages redeploys via `deploy.yml`. See `docs/rollback.md`.

**Restore images:** copy originals back into `assets/source/`, run `python3 scripts/process_images.py`, rebuild. Derivatives regenerate.

**Lost a data file:** restore it from Git history (`git checkout <sha> -- data/<file>.json`), verify against the authoritative source, rebuild.

## Integrity checks

- `tests/test_site.py` fails the build on broken links, privacy leaks, invalid JSON-LD, or private-report leakage.
- `build.py` exits non-zero on any missing asset/derivative or root-relative path.
- The public API `index.json` manifest lets a consumer detect a truncated/failed regeneration.

## Recommended cadence

Weekly automated artifact upload (via `research-intelligence.yml`) already snapshots `reports/` and `site/api/`. Add a monthly off-Git export of `data/` and a separate backup of `assets/source/` originals.
